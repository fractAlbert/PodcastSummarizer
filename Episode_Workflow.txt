EPISODE SUMMARY WORKFLOW
=========================

Generic workflow for processing any podcast episode.
Read this alongside the podcast's Podcast.config and Workflow.txt.

BEFORE STARTING
---------------
Read the podcast's Podcast.config for:
  - RSS feed URL
  - Episode file naming format
  - Transcript directory path


STEP 1 — SYNC EXISTING SUMMARIES FROM RSS FEED
------------------------------------------------
Fetch the RSS feed URL from Podcast.config.

Compare the descriptions for the last three published episodes against the
corresponding files in the Episode Summaries folder. Descriptions are sometimes
edited during publishing and the files may be out of date.

If any descriptions have changed, update the affected file(s).


STEP 2 — VERIFY THE LAST PROCESSED EPISODE
--------------------------------------------
Check the summary file for the most recently processed episode against the RSS
feed. If its description has been revised since it was saved, update the file.


STEP 3 — GENERATE THE TRANSCRIPT FOR THE NEW EPISODE
------------------------------------------------------
Run from the parent Podcast Summaries folder:
  python transcribe_episode.py "path\to\audio.mp3" "[TRANSCRIPT DIR]\[filename].txt"

Use the transcript directory and filename format from Podcast.config.

Requirements (one-time setup):
  pip install google-genai
  ffmpeg installed and in PATH (https://ffmpeg.org)
  GOOGLE_API_KEY environment variable set

If you already have a transcript, save it directly to the Transcripts folder.


STEP 4 — REVIEW THE TRANSCRIPT
--------------------------------
Read through the transcript and flag two types of problems:

1. Repeated content: passages where the same idea, sentence, or paragraph appears
   more than once in sequence. This can happen due to recording or editing errors
   and the repeated section should be removed. The wording may not be identical —
   look for the same point being made twice in a row.

2. Editing comments left in: statements that sound like production notes rather
   than spoken content, such as "should we cut this?", "leave that out", or
   "we can edit this part". These should be removed from the transcript.

If you find either type, create a brief summary of the issues found, including a
timestamp for each one. Then remove the offending text, save the corrected transcript,
and proceed.


STEP 5 — READ FORMAT RULES
----------------------------
Before writing anything, read Prompts/Description_Format.txt.
Do not skip this step — format rules vary per podcast.


STEP 6 — FOLLOW PODCAST-SPECIFIC STEPS
----------------------------------------
Read the podcast's Workflow.txt for any additional steps such as episode type
identification, voice identification, or show notes search conventions.
If the podcast's Workflow.txt says "no additional steps", proceed to Step 7.


STEP 7 — WRITE THE SUMMARY
----------------------------
Before writing, read 3–5 recent episode summaries from Episode Summaries/ to
calibrate tone, length, and level of detail. The goal is to match what is already
established — do not write more or less than is typical for this podcast.

Then write the episode summary following the rules in Description_Format.txt.
Use the hosts and any relevant details from Podcast.config.


STEP 8 — SAVE THE SUMMARY
---------------------------
Save to Episode Summaries/ using the file naming format from Podcast.config.


STEP 9 — COMMIT AND PUSH
--------------------------
Commit all new and updated files and push to GitHub.
The GitHub repo URL is in Podcast.config.
